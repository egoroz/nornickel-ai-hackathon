# «Скажи мне, кто твой шлиф»

Система классификации руд по OM-изображениям полированных шлифов, включая
гигапиксельные панорамы: сегментация фаз → типы срастаний → зоны
оталькования → класс руды + отчёт.

**Демо:** http://111.88.148.138:8501

## Как это работает

1. **Фазовая сегментация** (`src/sulfides.py`). На каждом изображении —
   кластеризация пикселей в осях (яркость, теплота R−B) и разметка кластеров
   правилами: сульфиды, серые фазы (магнетит), тёмные нерудные. Схема
   адаптивна к камере и экспозиции, поэтому отдельная нормализация
   изображений не нужна.
2. **Вкрапленники** (`src/intergrowth.py`). Оболочка зерна строится
   морфологическим замыканием сульфидной маски; по 10 масштабно-инвариантным
   признакам (степень замещения, раздробленность, толщина и др.) XGBoost
   относит зерно к обычным или тонким срастаниям. Близкие зёрна голосуют
   вместе — область получает единый вердикт.
3. **Тальк** (`src/talc.py`). U-Net (resnet18) сегментирует зоны скопления
   талька; запасной путь без torch — бустинг на ранговых/текстурных
   признаках. Внутри зон считаются тёмные вкрапления (вторая метрика).
4. **Правило класса** (`src/classify.py`): зоны талька > 10% площади →
   *оталькованная*; иначе по преобладающему типу срастаний → *рядовая* /
   *труднообогатимая*.
5. **Выход**: цветовой оверлей (зелёный — обычные, красный — тонкие,
   синий — тальк), карта классов по тайлам для панорам, таблица метрик,
   текстовое заключение, экспорт CSV / PDF / PNG-маски / labelme-JSON.

Панорамы обрабатываются тайлами 2048×2048 через pyvips без полной
декомпрессии; фазовая модель фитится один раз по уменьшенной копии, чтобы
тайлы сегментировались согласованно.

## Данные

Готовый датасет команды (чистка + разметка):
[Google Drive](https://drive.google.com/drive/folders/1A3UPKGtu2chogWpzSdCGbe2qY9vf6-Qk).
Скачанную папку положить рядом с проектом, в `data/dataset`:

```
data/dataset/
  photos/{ryadovaya,trudnoobogatimaya,otalkovannaya}/   фото + metadata.csv
  panorama_tiles/      тайлы панорам 2048×2048 + manifest.csv
  talc_annotation/     фото ч1 с исходной разметкой заказчика (labelme)
  talc_masks/          экспертные маски талька (PNG)
  train_inputs/        готовые входы табличного обучения (grains.csv, pixels.npz)
```

Ничего внутри не переименовывать и не перекладывать — пути используются
кодом и ноутбуками как есть. Изображения хранятся сырыми (без коррекции
яркости/контраста); вся вариативность съёмки закрывается инвариантными
признаками и аугментациями при обучении. Подробно про данные и их
подготовку — [EDA.md](EDA.md).

## Установка и запуск

```bash
python3.10 -m venv .venv
.venv/bin/pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
.venv/bin/pip install -r requirements.txt

.venv/bin/streamlit run app.py --server.maxUploadSize 1024   # интерфейс
.venv/bin/python -m src.pipeline --input <файл|папка> --out out/   # батч
```

Работает на CPU; при наличии CUDA сегментация талька автоматически
переезжает на GPU.

## Обучение

GPU-ноутбуки в `notebooks/` самодостаточны — рядом с ними должен лежать
только `data/dataset`:

| Ноутбук | Модель | Результат → куда положить |
|---|---|---|
| `datasphere_talc_unet.ipynb` | U-Net сегментации талька | `talc_unet.pt` → `artifacts/talc/unet.pt` |
| `datasphere_tabular.ipynb` | XGBoost: зёрна + тальк | `model_grains.joblib` → `artifacts/intergrowth/model.joblib`, `model_talc.joblib` → `artifacts/talc/model.joblib` |
| `datasphere_cnn_classifier.ipynb` | ConvNeXt-tiny (классификация фото) | `cnn_classifier.pt`, `cnn_metrics.json` → `artifacts/cnn/` |

То же обучение на своей машине с GPU: `deploy/vm_train_tabular.py`,
`deploy/vm_train_unet.py`.

Дообучение по циклу экспертной проверки: приложение экспортирует маску в
labelme-JSON → эксперт правит полигоны (`t`, флаг `no_talc` для «талька
нет») → `tools/labelme_to_masks.py` собирает маски → переобучение теми же
ноутбуками.

## Качество (hold-out по группам образцов, без утечек дублей)

| Задача | Метрика | Значение |
|---|---|---|
| Тип срастаний (по изображениям) | macro-F1 / AUC | 0.848 / 0.886 |
| Зоны талька (U-Net) | MAE доли / IoU | 3.9 п.п. / 0.66 |
| Тальк (пиксельный бустинг) | AUC | 0.946 |

Небольшое фото обрабатывается за 5–10 секунд, панорама — за 1–2 минуты;
работает на CPU, с GPU быстрее.

## Структура репозитория
